# 2. 기능명세

> 기준 문서: [1.요구사항 정리.md](./1.요구사항%20정리.md)  
> 요구사항: [1.1. 요구사항 - 백엔드.md](./1.1.%20요구사항%20-%20백엔드.md) · [1.2. 요구사항 - 프론트엔드.md](./1.2.%20요구사항%20-%20프론트엔드.md)

화면 와이어프레임(`money-app` · `PageController`)과 요구사항을 바탕으로 **구현 단위 기능명세**를 정의한다.  
등록·목록·상세·수정·삭제는 **기능마다 한 행**으로 분리 문서에 기술한다.

---

## 아키텍처

| 모듈 | 포트 | 역할 |
|------|------|------|
| `money-app` | 8080 | Thymeleaf 화면. 백엔드 API만 호출 |
| `money-backend-app` | 8081 | REST API (`/api/v1/*`, 헬스: `/api/v1/ha`) |

```
브라우저 → money-app (화면 URL)
              ↓ HTTP (JWT Bearer)
         money-backend-app (/api/v1/...)
              ↓
         data-mod / DB
```

## HTTP Method 규칙 (백엔드)

상세: [2.1. 기능명세 - 백엔드.md § HTTP Method 규칙](./2.1.%20기능명세%20-%20백엔드.md#http-method-규칙)

| Method | Path | Query | Body | 용도 |
|--------|:----:|:-----:|:----:|------|
| GET | ✅ or ✅ | ✅ or ✅ | ❌ | 조회 (Path·Query **혼용 ❌**, 예외: `StatisticsMonthlyGet`의 `view`) |
| POST | ❌ | ❌ | ✅ | 생성·로그인·업로드 |
| PATCH | ✅ | ❌ | ✅ (없음 가능) | **수정·upsert·상태 변경** |
| PUT | — | — | — | **미사용** |
| DELETE | ✅ | ❌ | ❌ | 삭제 |

- **PATCH**: Path=키, Body=변경분(omit=유지). **PUT은 사용하지 않음.**
- **GET 예외**: `StatisticsMonthlyGet`만 Path(`year`/`month`) + Query(`view?`) 혼용 허용. 상세는 [2.1](./2.1.%20기능명세%20-%20백엔드.md#http-method-규칙).

## 백엔드 구현 Phase (요약)

상세·API별 순서: [2.1 § 백엔드 구현 Phase](./2.1.%20기능명세%20-%20백엔드.md#백엔드-구현-phase)

| Phase | 범위 | API | 완료 기준 |
|-------|------|-----|-----------|
| **0** | 공통 (`common-mod`, `data-mod` 이전) | — | `{ resCode, data }` · 예외 · AOP |
| **1** | ① 회원·인증 | 1.1~1.16 | 로그인 → JWT·재발급 → `/members/me` |
| **2** | ② 수단 · ③ 지출유형 | 2.1~2.13 | 수단·유형 CRUD |
| **3** | ④ 지출·소득 | 3.1~3.12 | 지출·소득 CRUD + 할부 + 엑셀 일괄 |
| **4** | ④ 고정지출·가계부 | 4.1~4.8 | 월별 목록 + 고정지출 |
| **5** | ⑤ 목표금액 · ⑥ 통계 | 5.1~5.6 | 목표 목록·상세(연월 필수·이중 필드)·upsert + 통계 |
| **6** | 프론트 연동 | — | `money-app` → API 호출 ([2.2](./2.2.%20기능명세%20-%20프론트엔드.md)) |

**구현 순서:** Phase 0 → 1 → 2 → 3 → 4 → 5 → (6)
