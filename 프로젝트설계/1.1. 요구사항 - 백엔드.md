# 1.1. 요구사항 — 백엔드

> 기준 문서: [1.요구사항 정리.md](./1.요구사항%20정리.md) · 관련: [1.2. 요구사항 - 프론트엔드.md](./1.2.%20요구사항%20-%20프론트엔드.md)  
> 대상 모듈: `app-mod/money-backend-app`, `core-mod`, `data-mod`, `common-mod`

## 공통

- 모든 기능은 REST API로 제공한다.
- 응답 형식은 `{ resCode, data }`이며, 성공 시 `resCode`는 `0`이다.
- 검증·업무 오류는 정해진 에러 코드로 응답한다.
- 로그인한 사용자 본인 데이터만 조회·변경할 수 있다. (타인 가계부 API 없음)
- 관리자(1) / 일반(3) 권한에 따라 API 사용을 제한한다.
- 세션 기반 로그인을 사용한다.
- API 요청·응답은 AOP로 로그를 남긴다. (콘솔 + 파일)

**작성 규칙:** 등록·목록·상세 조회·수정·삭제는 **기능마다 한 행**으로 적는다. 여러 기능을 한 칸에 묶지 않는다.

### ① 회원 관리

| 기능 | 백엔드가 할 일 |
|------|----------------|
| 회원가입 | id 중복 확인, 비밀번호 규칙 검사, 닉네임 필수 저장, 권한 기본값 **일반(3)** 부여, **기본 지출유형 자동 등록** |
| 로그인 | id·비밀번호 확인, 비활성 계정이면 거부, 세션 생성 |
| 로그아웃 | 세션 종료 |
| id 찾기 | 가입 시 등록한 **이메일**로 id 조회 |
| 비밀번호 찾기 | id·닉네임이 모두 일치하면 비밀번호 변경 허용 (이메일 인증·보안질문 없음) |
| 본인 정보 수정 | 비밀번호(규칙 검사), 닉네임, 이메일, 폰, 소개 변경 |
| 비활성화 | 계정을 비활성 상태로 변경. 이후 로그인 불가. **이미 로그인 중인 세션은 끊지 않음** |
| 회원 추가 | **관리자만** 가능. 회원가입과 동일한 검사·기본 지출유형 자동 등록 |
| 회원 목록 | **관리자만** 가능. 페이징, id·닉네임 검색 |
| 다른 회원 정보 수정 | **관리자만** 가능. 비밀번호 변경 포함 |

**비밀번호 규칙:** 8자 이상, 영어 대·소문자·특수문자·숫자 중 3종류 이상  
**보안:** 비밀번호는 암호화해 저장. 일반 회원이 관리자 전용 API를 호출하면 거부.

### ② 지출수단·소득수단

지출수단과 소득수단은 **한 테이블**에 두고, **타입(지출/소득, 카드/계좌)** 으로 구분한다.

| 기능 | 백엔드가 할 일 |
|------|----------------|
| 등록 | 수단 이름, 카드/계좌, 사용 여부, 카드면 유효기간 저장. 로그인 사용자 소유로 저장 |
| 목록 (관리용) | 본인 수단 전체. 삭제 처리된 수단도 포함 |
| 목록 (지출·소득 입력용) | 본인 수단 중 **사용 여부=사용, 삭제되지 않은** 수단만 |
| 상세 조회 | 본인 수단 1건 조회 |
| 수정 | 본인 수단 1건의 이름·타입·사용 여부·유효기간 변경 |
| 삭제 | DB에서 지우지 않고 **삭제 표시만** 함. 이후 지출·소득 **새 등록 시 선택 목록에서 제외** |
| 과거 지출·소득 조회 | 수단을 삭제했어도, **이미 등록된 내역에 보이는 수단 이름은 등록 당시 그대로** 유지 |

### ③ 지출유형

| 기능 | 백엔드가 할 일 |
|------|----------------|
| 등록 | 본인 지출유형 추가 |
| 목록 | 본인 지출유형 목록 (기본 제공 + 사용자 추가) |
| 상세 조회 | 본인 지출유형 1건 조회 |
| 수정 | 이름, 사용/미사용, 아이콘 파일 저장 |
| 삭제 | **지출에 한 번이라도 쓰인 유형은 삭제 불가** → 오류 응답. 미사용 유형만 삭제 가능 |
| 기본 유형 자동 등록 | 회원가입·관리자 회원 추가 시 쇼핑·장보기 등 **기본 지출유형을 DB에 넣음** |
| 아이콘 파일 | 서버에 `{회원id}_{유형기본이름}.{확장자}` 로 저장. 기본 유형 아이콘 경로는 고정. 유형 삭제해도 **파일은 지우지 않음** |
| 지출 등록 시 | 지출 1건에 지출유형 **1개 필수** |

### ④ 가계부 (지출·소득·고정지출·엑셀)

#### 4-1. 지출·소득

| 기능 | 백엔드가 할 일 |
|------|----------------|
| 지출 등록 (일시불) | 수단·금액·결제일·장소·내용·지출유형 필수. 본인 수단·유형만 허용 |
| 지출 등록 (할부) | 할부 개월·월 납부액·시작 연월을 받아 **그 시점에 할부 건수만큼 지출 내역을 한꺼번에 생성**. 하나라도 실패하면 전부 취소 |
| 소득 등록 | 소득수단·금액·결제일 등 저장 |
| 지출 수정 | 지출 1건 수정. 할부로 생성된 건도 **건별** 수정 |
| 지출 삭제 | 지출 1건 삭제. 할부 중도상환은 **남은 할부 건 삭제**로 처리 |
| 소득 수정 | 소득 1건 수정 |
| 소득 삭제 | 소득 1건 삭제 |
| 할부 조건 변경 | 할부 개월·시작 연월 변경 API는 없음. **삭제 후 다시 등록** |

#### 4-2. 월별 가계부 목록

| 기능 | 백엔드가 할 일 |
|------|----------------|
| 월별 통합 목록 | 해당 연·월의 **일반 지출·소득 + 고정지출(아래 전개 규칙)** 을 **한 API**로 합쳐서 반환 |
| 할부 표시 | 조회 월에 해당하는 결제일의 할부 건만 포함 |
| 필터 | 지출수단, 지출유형, 결제일 범위, 내용·장소 부분 일치 검색 |
| 정렬 | 결제일 오름/내림차순, 지출수단, 지출유형 |

#### 4-3. 엑셀 일괄 등록

| 기능 | 백엔드가 할 일 |
|------|----------------|
| 양식 다운로드 | `.xlsx` 생성. 지출/소득 구분, 수단·지출유형 등 **셀에서 목록 선택** 가능하도록 작성 |
| 업로드 | `.xlsx`만, **최대 300행** |
| 등록 규칙 | **한 행이라도 오류면 전체 등록 취소**. 어느 행·항목이 잘못됐는지 응답 |
| 중복 | 같은 내용 여러 번 넣어도 **허용** |
| 소득 양식 | 지출과 **같은 컬럼 구성·검증 방식** |

#### 4-4. 고정지출

고정지출은 **매달 DB에 지출 내역을 따로 넣지 않는다.** 조회할 때만 해당 월 금액으로 **계산해 목록에 붙인다.**

| 기능 | 백엔드가 할 일 |
|------|----------------|
| 등록 | 이름(중복 가능), 수단, 금액, 매달 결제일(1~31일), 내용, 지출유형, **시작 연월**, **종료 연월** |
| 목록 (관리용) | 본인 고정지출 설정 목록. 페이징 |
| 상세 조회 | 고정지출 설정 1건 |
| 기본값 수정 | 앞으로 **월별 따로 바꾸지 않은 달**에 적용되는 기본 금액·결제일·내용 변경 |
| 월별 수정 | 특정 연·월만 다른 금액·결제일·내용 저장. 그 달 조회 시 **월별 값이 기본값보다 우선** |
| 삭제 | 고정지출 설정 삭제. **붙어 있던 월별 수정 데이터도 함께 삭제** |
| 월별 목록 반영 | 시작 연월 ~ 종료 연월 사이의 달만 가계부 목록에 나타남 |

### ⑤ 지출유형 목표금액

| 기능 | 백엔드가 할 일 |
|------|----------------|
| 기본 목표금액 등록 | 지출유형마다 목표금액 최초 설정 (0원 ~ 1억 원) |
| 기본 목표금액 수정 | 지출유형마다 기본 목표금액 변경 |
| 월별 목표금액 등록 | 특정 연·월·지출유형에 목표금액 최초 설정 |
| 월별 목표금액 수정 | 특정 연·월·지출유형의 목표금액 변경. 없으면 조회 시 기본 목표금액 사용 |

### ⑥ 월별 통계

| 기능 | 백엔드가 할 일 |
|------|----------------|
| 통계 계산 | 해당 월 지출·소득·고정지출(전개분)·할부를 합산 |
| 주별 지출 | **그 달 안에서만** 계산. **월요일**을 한 주의 시작으로 함. 1일이 월요일이 아니면 첫 주는 **1일부터** 합산 |
| 지출유형별 합계 | 금액 0원인 유형은 빼고 표시 |
| 목표 대비 사용률 | 그 달 목표금액(없으면 기본값) 대비 사용 %, 초과·적정·절약 판단 |
| 수단별 합계 | 0원인 수단도 표시 |
| 고정 vs 일반 | 고정지출 금액과 일반 지출 금액 비율 |
| 월 소득 합계 | 그 달 소득 총액 |
| 통계 저장 | 위 결과와 그때의 목표금액을 **그 달 통계로 DB에 저장** |
| 통계 조회 | 저장된 달 통계가 있으면 **저장본** 반환, 없으면 **지금 데이터로 계산한 값** 반환 |
| 통계 갱신 | 지출을 고쳐도 저장된 통계는 **자동으로 바뀌지 않음**. 사용자가 **다시 저장**할 때만 갱신 |
