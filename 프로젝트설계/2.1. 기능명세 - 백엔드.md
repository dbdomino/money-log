## 공통

- 모든 REST API는 **`/api/v1/`** prefix를 사용한다. (Breaking change 시 `/api/v2/` 등 추가)
- 응답 형식: `{ "resCode": 200, "data": { ... } }` (실패 시 4자리 `resCode`, `data`에 메시지)
- **목록 API** 응답: `data.list` = **JSON object 배열**. 상세: [기능명세/_공통.md § 목록 응답 규칙](기능명세상세-백엔드/_공통.md#목록-응답-규칙)
- **JWT + Refresh Token + DB 세션** 인증. 로그인 시 `accessToken`·`refreshToken` 발급·DB 저장. API마다 DB 대조. **중복 로그인 시 기존 세션 폐기**.
- **권한 = 로그인/관리자**인 API는 Access Token **만료·유효성 검증이 포함**된다 (공통 필터). 상세: [기능명세/_공통.md § 권한과 토큰 검증](기능명세상세-백엔드/_공통.md#권한과-토큰-검증-필수-규칙)
- 본인 데이터만 조회·변경. 관리자(1) / 일반(3) 권한으로 API 접근 제한.
- 요청·응답 AOP 로깅 (method, URI, 파라미터, resCode, 처리 시간)
- **기능 이름**: 사람이 읽는 한글 명칭 (예: 로그인 (토큰 발급)). **API 이름**: Controller·Client·문서 공통 식별자 (예: `MemberLogin`).
- HTTP Method별 Path·Query·Body 규칙은 아래 **「HTTP Method 규칙」** 을 따른다.

**작성 규칙:** 등록·목록·상세 조회·수정·삭제는 **기능마다 한 행**으로 적는다.

## 백엔드 구현 Phase

백엔드는 **Phase 0(공통) → Phase 1~5(API)** 순으로 구현한다.  
각 API의 **요청·응답 상세**는 [기능명세/](기능명세상세-백엔드/README.md) 디렉터리(1기능 1문서)를 따른다.

### API 1건 구현 시 반복 패턴

```
1. 기능명세/{phase}/ 문서 확인·보완
2. RequestDto / ResponseDto 정의 (문서와 동일 필드)
3. Repository → Service → Controller
4. Postman 요청·응답 예시를 문서 예시와 대조
```

## HTTP Method 규칙

`money-backend-app` 전 API가 지켜야 하는 **Path · Query · Body** 사용 규칙.

### 요약

| Method | Path | Query | Body | 용도 |
|--------|:----:|:-----:|:----:|------|
| **GET** | ✅ or ✅ | ✅ or ✅ | ❌ | 조회 |
| **POST** | ❌ | ❌ | ✅ | 등록·로그인·업로드 등 **생성** |
| **PATCH** | ✅ | ❌ | ✅ (없음 가능) | **수정·upsert·상태 변경** (부분 반영) |
| **PUT** | — | — | — | **현재 프로젝트에서 사용하지 않음** |
| **DELETE** | ✅ | ❌ | ❌ | 삭제 |

- ✅ or ✅ (GET): Path **또는** Query **중 하나만**. **동일 API에서 혼용 금지**.
  - **예외 (1개):** `StatisticsMonthlyGet` — Path `year`/`month` + Query `view?`(`saved`\|`live`). 모드 스위치용 Query만 허용.
- Path·Query·Body 칸의 ❌ = **해당 API에서 사용하지 않음**.
- **수정·upsert는 PATCH만** 사용한다. Body omit=유지 동작은 PATCH(부분 수정) 의미이므로 PUT 이름을 쓰지 않는다.

### GET — 조회

| 항목 | 규칙 |
|------|------|
| Body | **사용하지 않는다** |
| Path | 단건·복합키 조회 시 사용 (예: `/{expenseId}`, `/monthly/{year}/{month}`) |
| Query | 목록·필터·페이징 시 사용 (예: `?offset=0&limit=10&year=2026&month=7`) |
| Path vs Query | **한 API당 둘 중 하나만**. Path로 키를 주면 Query 추가하지 않음. **예외:** `StatisticsMonthlyGet`만 Path 키 + Query `view` 허용 |

**예**

```
GET /api/v1/expenses/{expenseId}              ← Path only
GET /api/v1/ledger/monthly?year=2026&month=7  ← Query only
GET /api/v1/statistics/monthly/2026/7         ← Path (저장본 우선)
GET /api/v1/statistics/monthly/2026/7?view=live  ← Path + Query view (예외)
```

### POST — 생성·실행

| 항목 | 규칙 |
|------|------|
| Body | **필수**. JSON 또는 `multipart/form-data` |
| Path | Path variable **사용하지 않는다** (리소스 id 없음) |
| Query | **사용하지 않는다** |

입력값·파일은 **전부 Body**에 담는다.

**예**

```
POST /api/v1/auth/login
Body: { "memberId", "password" }

POST /api/v1/expenses
Body: { "paymentMethodId", "amount", ... }
```

### PATCH — 수정·upsert·상태 변경

| 항목 | 규칙 |
|------|------|
| Path | 대상 **키**(PK·복합키·`/me` 등) |
| Body | **변경할 데이터**만 (변경분만 보내도 됨). Path 키를 Body에 **중복하지 않음** |
| Query | **사용하지 않는다** |

**Body 필드 처리**

| Body 값 | 서버 동작 |
|---------|-----------|
| 필드 **omit** (키 없음) | 해당 컬럼 **수정 안 함** |
| `"field": null` | 해당 컬럼 **비움** (nullable만) |
| `"field": 0` / `""` | **0 / 빈 문자열**로 저장 |

**용도:** 수정 폼 저장, 목표금액 upsert, 고정지출 월별 override, 회원 비활성화 등 **부분 반영**이 필요한 모든 변경.

**예**

```
PATCH /api/v1/payment-methods/{paymentMethodId}
Path: paymentMethodId
Body: { "name": "새 이름" }        ← 변경한 필드만. type 등 omit → 유지

PATCH /api/v1/admin/members/{memberId}/deactivate
Path: memberId
Body: (없음)                       ← 상태만 변경
```

### PUT — (미사용)

리소스 **전체 교체**(Body에 모든 필드 필수, omit 시 비움) 용도.  
현재 프로젝트 API에는 **PUT을 두지 않는다**. 부분 수정은 **PATCH**로 통일한다.

### DELETE — 삭제

| 항목 | 규칙 |
|------|------|
| Path | 삭제 대상 **키** |
| Body | **사용하지 않는다** |
| Query | **사용하지 않는다** |

**예**

```
DELETE /api/v1/expenses/{expenseId}
Path: expenseId
```

## PATCH (전체)

**Path = 대상 키 · Body = 변경 데이터(변경분만 가능) · Query 없음**

### Body 필드 처리

| Body 값 | 서버 동작 |
|---------|-----------|
| 필드 **omit** | 해당 컬럼 **수정 안 함** |
| `"field": null` | 해당 컬럼 **비움** (nullable만) |
| `"field": 0` / `""` | **0 / 빈 문자열**로 저장 |

### API별 Path · Body

| API 이름 | Method | Path (키) | Request Body (변경 데이터) |
|----------|--------|-----------|---------------------------|
| `MemberUpdateMe` | PATCH | (고정 `/members/me`) | `password?`, `nickname?`, `email?`, `phone?`(하이픈 없이 숫자만), `intro?` |
| `AdminMemberUpdate` | PATCH | `memberId` | `password?`, `nickname?`, `role?`, `email?`, `phone?`(하이픈 없이 숫자만), `intro?` |
| `AdminMemberDeactivate` | PATCH | `memberId` | (없음) |
| `PaymentMethodUpdate` | PATCH | `paymentMethodId` | `name?`, `type?`, `inUse?`, `cardExpiry?` |
| `ExpendGroupUpdate` | PATCH | `expendGroupId` | `name?`, `inUse?`, `iconFile?` (multipart) |
| `ExpenseUpdate` | PATCH | `expenseId` | `paymentMethodId?`, `amount?`, `paymentDate?`, `place?`, `content?`, `expendGroupId?` |
| `IncomeUpdate` | PATCH | `incomeId` | `paymentMethodId?`, `amount?`, `paymentDate?`, `content?` |
| `FixedExpenseUpdate` | PATCH | `fixedExpenseId` | `name?`, `paymentMethodId?`, `amount?`, `paymentDayOfMonth?`, `content?`, `expendGroupId?`, `startYearMonth?`, `endYearMonth?` |
| `FixedExpenseMonthlyOverrideUpsert` | PATCH | `fixedExpenseId`, `year`, `month` | `amount?`, `paymentDayOfMonth?`, `content?` |
| `ExpendTargetDefaultUpsert` | PATCH | `expendGroupId` | `defaultTargetAmount` |
| `ExpendTargetMonthlyUpsert` | PATCH | `year`, `month`, `expendGroupId` | `monthlyTargetAmount` |
| `ExpenseSettleInstallmentRemainder` | PATCH | `installmentGroupId` | (없음·빈 `{}` 허용) |

`?` = omit 가능. 보낸 필드만 갱신.
