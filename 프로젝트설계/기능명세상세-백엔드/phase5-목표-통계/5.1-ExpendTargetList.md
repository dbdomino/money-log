### 성공 (`resCode: 200`) — `data`

| 필드 | 타입 | 설명 |
|------|------|------|
| `year` | number | 조회 연도. 요청 `year`와 동일 |
| `month` | number | 조회 월. 요청 `month`와 동일 |
| `list` | array (object[]) | **사용 중** 유형별 목표 (현재 페이지) |
| `list[].expendGroupId` | number | 지출유형 PK |
| `list[].expendGroupName` | string | 지출유형 이름 |
| `list[].defaultTargetAmount` | number | **기본(템플릿)** 월 목표금액 (0~100,000,000). 미설정 시 `0` |
| `list[].monthlyTargetAmount` | number \| null | 요청 `year`/`month`에 **별도 저장된** 월별 목표. 없으면 `null` (기본값을 복사하지 않음) |
| `offset` | number | 이번 조회에서 건너뛴 건수. 요청 `offset`과 동일(보정 시 보정값) |
| `limit` | number | 이번 조회에서 가져온 최대 건수. 요청 `limit`과 동일(보정 시 보정값) |
| `totalCount` | number | 검색 조건에 맞는 **전체 건수**. **사용 중 유형만** 집계 |

### 실패 — 대표 `resCode`

| resCode | 조건 |
|---------|------|
| 1001 | 로그인 필요 |
| 3603 | `year`/`month` 누락, 또는 월 범위 오류 (1~12) |
| 9001 | offset·limit 누락·범위 오류 |

## 비고

- **연·월 필수.** 기본 목표만 보려는 화면도 조회 연·월(예: 이번 달)을 넘긴다. 모드 분기 없음.
- `defaultTargetAmount`와 `monthlyTargetAmount`를 **별도 필드**로 내려 저장 여부를 구분한다. 통계 등 **적용 금액**은 `monthlyTargetAmount ?? defaultTargetAmount`.
- **사용 중(`inUse=true`) 유형만** 반환. 사용 여부 변경은 `ExpendGroupCreate`/`ExpendGroupUpdate`.
- 기본 목표 저장: [`ExpendTargetDefaultUpsert`](./5.3-ExpendTargetDefaultUpsert.md). 해당 달 목표 저장: [`ExpendTargetMonthlyUpsert`](./5.4-ExpendTargetMonthlyUpsert.md). 단건 확인: [`ExpendTargetGet`](./5.2-ExpendTargetGet.md).
- 목표금액은 **월 단위** 한도 (0 ~ 1억 원).
- Path·Query **혼용 없음** (Query only).
