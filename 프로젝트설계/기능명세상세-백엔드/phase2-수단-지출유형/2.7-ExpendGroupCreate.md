## 응답

공통 래퍼: [_공통.md § 응답 래퍼](../_공통.md#응답-래퍼)  
공유 타입: [_공통.md § ExpendGroup](../_공통.md#공유-타입--expendgroup)

### 성공 (`resCode: 200`) — `data`

| 필드 | 타입 | 설명 |
|------|------|------|
| `expendGroupId` | number | PK |
| `name` | string | 유형 이름 |
| `inUse` | boolean | |
| `iconUrl` | string \| null | `/api/v1/expend-groups/icons/{filename}` |
| `defaultGroup` | boolean | `false` (사용자 생성) |

### 실패 — 대표 `resCode`

| resCode | 조건 |
|---------|------|
| 1001 | 로그인 필요 |
| 3101 | 이름 중복 (본인 내) |
| 3102 | 아이콘 파일 형식·크기 오류 |
| 9001 | 필수 필드 누락 |

## 예시

### 성공

**Request**

```http
POST /api/v1/expend-groups HTTP/1.1
Host: localhost:8081
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Content-Type: multipart/form-data; boundary=----Boundary

------Boundary
Content-Disposition: form-data; name="name"

취미
------Boundary
Content-Disposition: form-data; name="inUse"

true
------Boundary
Content-Disposition: form-data; name="iconFile"; filename="hobby.png"
Content-Type: image/png

(binary)
------Boundary--
```

**Response**

```json
{
  "resCode": 200,
  "data": {
    "expendGroupId": 10,
    "name": "취미",
    "inUse": true,
    "iconUrl": "/api/v1/expend-groups/icons/user01_취미.png",
    "defaultGroup": false
  }
}
```

### 실패

```json
{
  "resCode": 3102,
  "data": {
    "message": "지원하지 않는 아이콘 파일 형식입니다"
  }
}
```

## 비고

- 아이콘 저장명: `{회원id}_{유형기본이름}.{확장자}`
- 응답 `iconUrl`은 바이너리가 아니라 조회 경로. 프론트 수신 방법: [_공통 § 지출유형 아이콘 · 프론트 수신](../_공통.md#지출유형-아이콘--프론트-수신).
- 회원가입 시 생성되는 **기본 지출유형**은 `defaultGroup=true`.
- `iconFile` 생략 시 `iconUrl=null`.
