## 응답

공통 래퍼: [_공통.md § 응답 래퍼](../_공통.md#응답-래퍼)  
인증: [_공통.md § Refresh Token](../_공통.md#refresh-token) · [_공통.md § 토큰 검증](../_공통.md#토큰-검증-공통--로그인-필요-api)

### 성공 (`resCode: 200`) — `data`

| 필드 | 타입 | 설명 |
|------|------|------|
| `accessToken` | string | 새 JWT Access Token |
| `tokenType` | string | `Bearer` |
| `expiresIn` | number | Access Token 만료까지 남은 시간(초). 기본 86400(1일) |
| `refreshToken` | string | 새 Refresh Token (**Rotation**) |
| `refreshExpiresIn` | number | Refresh Token 만료까지 남은 시간(초). 기본 604800(7일) |

### 실패 — 대표 `resCode`

| resCode | 조건 |
|---------|------|
| 1005 | Refresh Token 없음·만료·해시 불일치 |
| 1006 | 세션 폐기됨 (다른 곳에서 로그인·로그아웃) |
| 1004 | 비활성 계정 |
| 9001 | 필수 필드 누락 |

## 예시

### 성공

**Request**

```http
POST /api/v1/auth/refresh HTTP/1.1
Host: localhost:8081
Content-Type: application/json

{
  "refreshToken": "dGhpcy1pcy1hLXJlZnJlc2gtdG9rZW4..."
}
```

**Response**

```json
{
  "resCode": 200,
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "tokenType": "Bearer",
    "expiresIn": 86400,
    "refreshToken": "bmV3LXJlZnJlc2gtdG9rZW4...",
    "refreshExpiresIn": 604800
  }
}
```

### 실패 (세션 폐기 — 중복 로그인)

```json
{
  "resCode": 1006,
  "data": {
    "message": "다른 곳에서 로그인되어 세션이 종료되었습니다"
  }
}
```

### 실패 (Refresh 만료)

```json
{
  "resCode": 1005,
  "data": {
    "message": "유효하지 않거나 만료된 토큰입니다"
  }
}
```

## 비고

- Access Token(1일) 만료 시 **재로그인 없이** 새 토큰 쌍을 발급한다.
- DB에서 `refresh_token_hash`·`refresh_expires_at`·`revoked=false`를 검증한다. ([_공통 § Refresh Token 검증](../_공통.md#토큰-검증-공통--로그인-필요-api))
- **Rotation**: 성공 시 동일 `session_id` 행의 Access·Refresh 해시와 만료 시각을 **갱신**한다.
- 비활성(`active=false`) 계정은 `1004`.
