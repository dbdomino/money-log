# 1.2. 요구사항 — 프론트엔드

> 기준 문서: [1.요구사항 정리.md](./1.요구사항%20정리.md) · 관련: [1.1. 요구사항 - 백엔드.md](./1.1.%20요구사항%20-%20백엔드.md)  
> 대상 모듈: `app-mod/money-app` (Thymeleaf)  
> 데이터는 **백엔드 API로만** 가져온다. DB에 직접 접근하지 않는다.

## 공통

- 로그인 상태는 백엔드 세션·쿠키와 맞춘다.
- API가 실패하면 `resCode`·메시지를 화면에 보여 준다.
- 로그인하지 않으면 로그인 화면으로 보낸다.
- **일반 회원**에게는 회원 관리 메뉴를 보이지 않게 한다.
- **본인 가계부 화면만** 만든다. (타인 가계부 화면 없음)

**작성 규칙:** 등록·목록·상세 조회·수정·삭제는 **기능마다 한 행**으로 적는다. 여러 기능을 한 칸에 묶지 않는다.

---

### ① 회원 관리

| 기능 | 프론트엔드가 할 일 |
|------|-------------------|
| 회원가입 | id, 비밀번호, 비밀번호 확인, 닉네임(필수), 이메일·폰·소개(선택) 입력 화면. 비밀번호 규칙 안내 |
| 로그인 | id·비밀번호 입력. 실패·비활성 계정 안내 |
| 로그아웃 | 로그아웃 후 로그인 화면으로 이동 |
| id 찾기 | 이메일 입력 → 조회된 id 표시 |
| 비밀번호 찾기 | id·닉네임 입력 → 일치하면 새 비밀번호 입력 화면 |
| 본인 정보 수정 | 비밀번호, 닉네임, 이메일, 폰, 소개 수정 화면 |
| 회원 목록 | **관리자만** 메뉴·화면 노출. 목록·페이징·id/닉네임 검색 |
| 회원 추가 | **관리자만** 목록에서 신규 회원 등록 |
| 다른 회원 수정 | **관리자만** 회원 선택 후 정보·비밀번호 변경 |
| 비활성화 | **관리자만** 회원을 비활성 처리 |
| 권한 없는 접근 | 관리자 전용 URL 직접 입력 시 오류·로그인/메인으로 이동 |

**비밀번호 안내 문구:** 8자 이상, 영어 대·소문자·특수문자·숫자 중 3종류 이상

### ② 지출수단·소득수단

| 기능 | 프론트엔드가 할 일 |
|------|-------------------|
| 등록 | 카드/계좌, 이름, 사용 여부 입력. 카드면 유효기간 입력란 표시 |
| 목록 (관리) | 본인 수단 목록 테이블 |
| 상세 조회 | 1건 읽기 전용 화면 |
| 수정 | 수정 폼에서 변경 후 저장 |
| 삭제 | 확인 후 삭제 요청. **과거 지출·소득에 적힌 수단 이름은 그대로 남는다**는 안내 |
| 목록 (지출·소득 입력) | 지출·소득 등록 화면의 수단 선택 목록 (**사용 중인 수단만**) |

---

### ③ 지출유형

| 기능 | 프론트엔드가 할 일 |
|------|-------------------|
| 등록 | 이름, 사용/미사용, 아이콘 파일 선택·미리보기 |
| 목록 | 기본·사용자 유형 목록, 아이콘 이미지 표시 |
| 상세 조회 | 1건 읽기 전용 |
| 수정 | 이름·사용 여부·아이콘 변경 |
| 삭제 | 확인 후 요청. **이미 쓰인 유형이면** 서버 오류 메시지를 알림으로 표시하고 **비활성화 안내** |
| 지출 입력 | 지출유형 **하나 필수** 선택. 미사용 유형은 선택 불가 |
| 아이콘 | 가계부·통계 화면에서 유형 아이콘 표시 |

---

### ④ 가계부 (지출·소득·고정지출·엑셀)

#### 4-1. 지출·소득 입력

| 기능 | 프론트엔드가 할 일 |
|------|-------------------|
| 지출 등록 | 일시불/할부 선택(기본 일시불), 수단·금액·날짜·장소·내용·지출유형 입력 |
| 할부 입력 | 할부 선택 시 개월 수, 월 납부액, 시작 연월 입력란 표시 |
| 소득 등록 | 소득수단, 금액, 결제일 등 입력 |
| 지출 수정 | 지출 1건 수정 폼. 할부 건도 **건별** 수정 |
| 지출 삭제 | 지출 1건 삭제 확인 |
| 소득 수정 | 소득 1건 수정 폼 |
| 소득 삭제 | 소득 1건 삭제 확인 |
| 할부 안내 | 개월·시작 연월을 바꾸려면 **삭제 후 다시 등록** 안내 문구 |
| 중도상환 | 남은 할부 건을 선택해 삭제 (이후 버전에서 전용 화면 가능) |

#### 4-2. 월별 가계부 목록

| 기능 | 프론트엔드가 할 일 |
|------|-------------------|
| 월 선택 | 연·월 선택 (기본: 이번 달) |
| 통합 목록 | 백엔드 **월별 통합 목록 API** 결과를 **한 테이블**에 표시 (일반 지출·소득 + 고정지출) |
| 구분 | 일반 지출 / 소득 / 고정지출 / 할부 등을 컬럼 또는 뱃지로 구분 |
| 필터 | 수단, 지출유형, 날짜 범위, 내용·장소 검색 |
| 정렬 | 결제일, 수단, 지출유형 |
| 합계 | 그 달 지출·소득 합계 표시 (API에서 받은 값 사용) |

#### 4-3. 엑셀 일괄 등록

| 기능 | 프론트엔드가 할 일 |
|------|-------------------|
| 양식 받기 | 버튼으로 `.xlsx` 다운로드 |
| 업로드 | `.xlsx` 선택. 300행 초과 시 업로드 전 경고 |
| 결과 | 성공 건수 또는 **몇 행·어느 항목이 틀렸는지** 표시 |
| 중복 | 중복 행도 그대로 등록. 별도 경고 없음 |

#### 4-4. 고정지출

| 기능 | 프론트엔드가 할 일 |
|------|-------------------|
| 등록 | 이름, 수단, 금액, **매달 결제일**, 내용, 지출유형, **시작 연월**, **종료 연월** 입력 |
| 목록 (관리) | 고정지출 설정 목록 (가계부 월 목록과 **별도 화면**), 페이징 |
| 상세 조회 | 설정 1건 읽기 전용 |
| 기본값 수정 | 템플릿 금액·결제일·내용 변경. **월별로 따로 안 바꾼 달**에도 반영된다는 안내 |
| 월별 수정 | 연·월을 고른 뒤 **그 달만** 금액·결제일·내용 변경 |
| 삭제 | 확인 후 삭제 |
| 가계부 | 월 목록을 열면 기간 안의 고정지출이 **자동으로 함께** 보임 |

### ⑤ 지출유형 목표금액

| 기능 | 프론트엔드가 할 일 |
|------|-------------------|
| 기본 목표금액 등록 | 지출유형 설정에서 유형별 목표금액 최초 입력 (0 ~ 1억) |
| 기본 목표금액 수정 | 지출유형 설정에서 유형별 목표금액 변경 |
| 월별 목표금액 등록 | 통계 화면에서 **그 달·그 유형** 목표금액 최초 입력 |
| 월별 목표금액 수정 | 통계 화면에서 **그 달·그 유형** 목표금액 변경. 비우면 기본값 표시 |

---

### ⑥ 월별 통계

| 기능 | 프론트엔드가 할 일 |
|------|-------------------|
| 월 선택 | 통계를 볼 연·월 |
| 주별 지출 | 표 또는 차트로 주간 합계 표시 |
| 지출유형별 | 0원 유형은 숨김 |
| 목표 대비 | 사용 %, 초과·적정·절약 문구·색상 |
| 수단별 | 0원 수단 포함 전체 표시 |
| 고정 vs 일반 | 비율 숫자 또는 차트 |
| 월 소득 | 그 달 소득 총액 |
| 통계 저장 | 「통계 저장」 버튼 → 저장 API 호출 |
| 조회 | 저장된 통계가 있으면 **저장본**, 없으면 **지금 계산한 값** 표시 |
| 갱신 안내 | 지출을 바꿔도 저장된 통계는 안 바뀜 → **다시 저장해야 반영** 안내 |
