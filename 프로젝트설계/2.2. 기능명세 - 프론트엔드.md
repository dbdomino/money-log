## 공통

- DB 직접 접근 없음. 모든 데이터는 **백엔드 API** (`/api/v1/*`)로 조회·변경한다.
- 로그인 API 응답의 토큰을 저장한다. API 호출 시 `Authorization: Bearer`. **로그인/관리자 API는 서버가 토큰 만료·유효성을 검증**한다 (공통 필터). 앱 기동·보호 화면 진입 시 `MemberTokenValidate`로 유효성 확인. **1001/1006** 시 로그인 화면으로 이동 (다른 곳 로그인 안내). Access 만료(`1001`) 시 `MemberTokenRefresh` 시도 후 실패하면 로그인.
- API 실패 시 `resCode`·메시지를 화면(alert 또는 폼 상단)에 표시한다.
- **POST** API 호출 시 입력값은 **Request Body(JSON/multipart)** 로만 전송한다. Query·Path 파라미터 사용하지 않는다.
- **GET** API 호출 시 **Body 없음**. Path **또는** Query **중 하나만** 사용한다 (혼용 금지). **예외:** `StatisticsMonthlyGet`만 Path 연·월 + Query `view?`(`saved`\|`live`) 허용.
- **페이징 목록** (`AdminMemberList`, `FixedExpenseList` 등): Query `offset`·`limit`으로 요청. 응답의 `offset`·`limit`·`totalCount`로 현재 페이지·전체 페이지 수를 계산한다 ([기능명세/_공통.md § 페이징](기능명세상세-백엔드/_공통.md#공유-타입--페이징)).
- **PATCH**: Path(대상 키) + Body(변경 데이터). 변경한 필드만 Body에 담는다. **PUT은 사용하지 않음.**
- **DELETE**: Path(대상 키)만. Body·Query 없음.
- 미로그인 시 **로그인 필수** URL → `/auth/login` redirect. **비로그인 허용:** 화면 1.1~1.6 (`/auth/*`, `/error/forbidden`).
- **일반 회원**에게 회원 관리 메뉴·URL 비노출. 관리자 URL 직접 접근 → `/error/forbidden`.
- **본인 가계부 화면만** 제공 (타인 가계부 없음).

**작성 규칙:** 등록·목록·상세 조회·수정·삭제는 **기능마다 한 행**으로 적는다.  
**상세 문서:** 화면 구성·동작·검증은 [기능명세상세-프론트엔드](기능명세상세-프론트엔드/README.md)에 **화면마다 1문서**로 작성한다. 이 파일(2.2)은 개요·색인용이다.  
**표시방식:** 목록·허브·auth = **페이지**, 등록·수정·상세 = **모달**(부모 URL + `?m=`). 기획: [화면기획/](화면기획/README.md).

### 레이아웃

| 레이아웃 | 용도 | 템플릿 |
|----------|------|--------|
| auth | 로그인·가입·찾기 | `layout/auth.html` |
| main | 로그인 후 (사이드바·상단바) | `layout/main.html` |

### 표시방식

| 구분 | URL | 템플릿 |
|------|-----|--------|
| 페이지 | 대표 경로만 | `templates/**/*.html` + layout |
| 모달 | 부모 + `?m=` (+ `id`) | 부모 페이지에 fragment include |

구 URL(`/payments/new` 등) → 부모 딥링크로 redirect.

### 내부 구조 (구현 가이드)

```
money-app
├── controller/PageController     # GET 페이지 (와이어프레임)
├── web/*WebController            # (추가) POST 폼 → API Body → redirect
└── client/BackendApiClient       # :8081/api/v1 호출, POST는 JSON/multipart body
```
