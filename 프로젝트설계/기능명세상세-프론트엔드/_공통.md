## 필드 설명 작성 규칙 (누락)

화면 상세 문서의 표(입력·표시·동작) **설명 칸은 그 칸만 보고 의미가 읽혀야** 한다.

- `_공통.md`에 같은 내용이 있어도 **화면 문서 표에 의미를 다시 적는다**.
- 다음만 적는 것은 **누락**이다: `동일`, `위와 같음`, `공통 참고`, `생략`, 빈 칸, `요청 에코`.
- 링크는 **보조**일 뿐, 설명 칸을 대체하지 않는다.

## 표시방식 (페이지 / 모달)

| 구분 | 대상 | 표시 | URL |
|------|------|------|-----|
| **페이지** | 목록·허브·auth·본인정보·엑셀·목표·통계 | 풀페이지 (`templates/**/*.html`) | 대표 URL만 |
| **모달** | 등록·수정·상세 (및 고정지출 월별) | 부모 페이지 위 모달 (`fragments` / `th:fragment`) | 부모 URL + `?m=` (+ `id` 등) |

규칙:

- 사이드바「지출 등록」「소득 등록」「고정지출 등록」은 **전용 페이지가 아니라** 부모로 이동 후 모달 오픈.
- 구 경로(`/payments/new`, `/ledger/expenses/new` 등)는 구현 시 **부모 + `?m=`로 redirect**.
- 삭제·정지·중도상환 확인은 공통 `fragments/confirm-dialog`.
- 기획·프로토: [화면기획/01-화면구성.md](../화면기획/01-화면구성.md)

모달 딥링크 예:

| `?m=` | 의미 | 부모 |
|-------|------|------|
| `create` / `detail` / `edit` | CRUD | 해당 목록 (수단·유형·회원·고정지출) |
| `expense-create` / `expense-edit` | 지출 | `/ledger` (5.1) |
| `income-create` / `income-edit` | 소득 | `/ledger` (5.1) |
| `monthly` | 고정지출 월별 | `/fixed-expenses` (4.2) |

## 인증·토큰 (화면 공통)

로그인 API 응답의 토큰을 저장한다. API 호출 시 `Authorization: Bearer <accessToken>`.

| 시점 | 처리 | 호출 API |
|------|------|----------|
| 로그인 성공 | `accessToken`·`refreshToken` 저장 | `MemberLogin` |
| 앱 기동·보호 화면 진입 | 저장 토큰 유효성 확인 | `MemberTokenValidate` |
| Access 만료 (`resCode` 1001) | 재발급 시도 → 실패 시 로그인 | `MemberTokenRefresh` |
| 다른 곳 로그인·세션 무효 (`1001`/`1006`) | 토큰 삭제 후 로그인 화면 | — |
| 로그아웃 | JWT 비활성화 후 토큰 삭제·로그인 이동 | `MemberTokenRevoke` |

### 로그인 필요 여부

| 구분 | 화면번호 | 화면 |
|------|----------|------|
| **로그인 불필요** | 1.1~1.6 | 로그인, 회원가입, 아이디 찾기, 비밀번호 찾기, 비밀번호 변경, 권한 없음 |
| **로그인 필수** | 1.7 및 Phase 2~6 전부 | 본인 정보, 수단·유형·가계부·고정지출·목표·통계 등 |
| **관리자 필수** | 1.8~1.10 | 회원 목록·추가·수정 |

- 미로그인 시 **로그인 필수** URL → `/auth/login` redirect.
- **일반 회원**에게 회원 관리 메뉴·URL 비노출. 관리자 URL 직접 접근 → `/error/forbidden` (1.6, 로그인 불필요).
- **본인 가계부 화면만** 제공 (타인 가계부 없음).

토큰 API 상세: [기능명세상세-백엔드 phase1](../기능명세상세-백엔드/phase1-회원/_인증-토큰.md)

## API 호출 규칙

| Method | Path | Query | Body | 프론트 규칙 |
|--------|:----:|:-----:|:----:|-------------|
| GET | ✅ or ✅ | ✅ or ✅ | ❌ | Path **또는** Query **중 하나만** (혼용 금지) |
| POST | ❌ | ❌ | ✅ | Body만 (JSON/multipart) |
| PATCH | ✅ | ❌ | ✅ | Path(키) + Body(변경분만). **PUT 미사용** |
| DELETE | ✅ | ❌ | ❌ | Path만 |

**예외:** `StatisticsMonthlyGet`만 Path 연·월 + Query `view?`(`saved`\|`live`) 허용.

### 실패 표시

- API 실패 시 `resCode`·`data.message`를 화면(alert 또는 폼 상단)에 표시한다.
- 컨트롤러에서 try-catch로 응답을 제각각 만들지 말고, 공통 클라이언트·에러 처리에 맡긴다.

### 페이징 UI

요청: Query `offset`·`limit`.  
응답: `offset`·`limit`·`totalCount`로 현재 페이지·전체 페이지 수 환산.

환산식: [기능명세상세-백엔드/_공통.md § 페이징](../기능명세상세-백엔드/_공통.md#공유-타입--페이징)
