# 인증·토큰 (공통)

> Phase 1 · 로그인 직후 **토큰 발급 → 갱신 → 로그아웃** 흐름  
> 공통 규칙: [_공통.md § 인증](../_공통.md#인증)

## 용어 (혼동 방지)

| 용어 | 의미 | 관련 API |
|------|------|----------|
| **토큰 발급** | 로그인 성공 시 Access·Refresh + DB 해시 저장 | `MemberLogin` |
| **토큰 갱신** | Refresh로 새 JWT 쌍 발급 (DB 해시 갱신) | `MemberTokenRefresh` |
| **로그아웃 (JWT Token 비활성화)** | 사용자 로그아웃. DB 해시 NULL. Access만으로 호출 | `MemberTokenRevoke` |

> 관리자 회원 정지(`AdminMemberDeactivate`)는 [14-AdminMemberDeactivate.md](./14-AdminMemberDeactivate.md) — **본 흐름과 별개**.

## API 흐름

```
MemberLogin (3)        → 토큰 발급, DB 해시 저장
       ↓
MemberTokenRefresh (4) → 토큰 갱신
       ↓
MemberTokenRevoke (5)  → 로그아웃 (JWT Token 비활성화)
```

**중복 로그인:** 새 로그인 시 기존 세션 **JWT 비활성화** → 이전 기기 `1006`/`1005`.
