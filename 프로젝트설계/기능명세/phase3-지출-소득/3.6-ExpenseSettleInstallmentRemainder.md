| 항목 | 내용 |

## 요청

### Path Variables

| 이름 | 타입 | 필수 | 설명 |
|------|------|:----:|------|
| `installmentGroupId` | number | ✅ | 중도상환할 할부 그룹 id |

### Query Parameters

없음

### Headers

| 이름 | 필수 | 설명 |
|------|:----:|------|
| `Authorization` | ✅ | `Bearer <accessToken>` |
| `Content-Type` | ❌ | Body를 보낼 경우 `application/json` |

### Body

없음 (처리할 대상은 Path의 `installmentGroupId`만으로 충분). 빈 `{}`를 보내도 된다.

## 응답

공통 래퍼: [_공통.md § 응답 래퍼](../_공통.md#응답-래퍼)

### 성공 (`resCode: 200`) — `data`

| 필드 | 타입 | 설명 |
|------|------|------|
| `installmentGroupId` | number | 중도상환 처리한 할부 그룹 id |
| `settledCount` | number | 중도상환으로 제거된 **남은** 할부 건수 |
| `message` | string | 예: `남은 할부 9건이 중도상환 처리되었습니다` |

### 실패 — 대표 `resCode`

| resCode | 조건 |
|---------|------|
| 1001 | 로그인 필요 |
| 3206 | 할부 그룹 없음 |
| 3207 | 처리할 남은 할부 건 없음 |

## 비고

- **중도상환**: 할부 그룹의 남은 스케줄을 정리하는 **상태·일정 처리**다. 단건 자원 삭제가 아니므로 `DELETE`가 아니라 `PATCH`를 쓴다.
- 동작: 오늘 이후(또는 미결제) **남은 할부 지출 건을 일괄 제거**한다. 이미 지난 월 할부 건은 유지.
- 본인 할부 그룹만 처리.
- 이전 API 이름 `ExpenseDeleteInstallmentRemainder`(DELETE)는 본 명세로 대체.
