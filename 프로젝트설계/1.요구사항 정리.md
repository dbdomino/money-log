# 1. 요구사항 정리

## 공통 원칙

- 모든 가계부 데이터(지출·소득·수단·지출유형·고정지출·통계)는 **로그인한 본인 것만** 조회·등록·수정·삭제할 수 있다.
- 타인 가계부 조회는 **다음 버전**에서 관리자 전용으로 추가 예정이며, **현재 버전에는 반영하지 않는다.**
- **작성 규칙:** 등록·목록·상세 조회·수정·삭제는 **기능마다 한 줄(한 항목)** 로 적는다.

---

### ① 회원 관리 기능

#### 회원가입·로그인

- 회원가입
  - 필수: id, pw, 닉네임
  - 선택: 이메일주소, 폰번호, 소개
  - 가입 시 권한 기본값: **일반(3)**
- 로그인·**토큰 갱신**·**로그아웃 (JWT Token 비활성화)** (**JWT + Refresh Token**, **DB 세션 관리**)
  - Access Token 만료: 1일
  - Refresh Token 만료: 7일
  - **회원당 활성 세션 1개** — 중복 로그인 시 기존 세션 JWT 비활성화
  - API 요청마다 DB에 저장된 토큰 해시·만료 시각과 대조해 유효성 검증
- 비밀번호 정책
  - 8자 이상
  - 영어 대문자·영어 소문자·특수문자·숫자 중 **3종류 이상** 포함

#### 계정 찾기·정보 변경

- id 찾기: 가입 시 등록한 **이메일**로 id 조회
- 비밀번호 찾기
  - id와 닉네임이 일치하면 비밀번호 변경 가능
  - 이메일 인증·보안질문은 사용하지 않음
- 회원정보 수정 (본인)
  - pw, 닉네임, 이메일주소, 폰번호, 소개

#### 회원 비활성화

- 회원 탈퇴 기능은 제공하지 않음
- 비활성화 기능 제공
  - 비활성화된 계정은 로그인 불가
  - 기존 세션·토큰 강제 폐기. 폐기된 토큰으로 API 호출 시 거부

#### 권한 모델

| 권한 | 코드 | 설명 |
|------|------|------|
| 관리자 | 1 | 회원 관리 전용 권한 |
| 일반 | 3 | 본인 가계부만 사용 |

- **관리자(1)** 만 가능한 기능
  - 회원 추가 (회원 목록 화면에서 등록)
  - 회원 목록 조회 (페이징, id 또는 닉네임 검색)
  - 다른 회원 정보 설정 변경 (비밀번호 변경 포함)
  - 회원 비활성화
- **일반(3)** 회원
  - 회원 목록 조회·회원 추가 불가
  - 다른 회원 정보 변경 불가
  - 본인 가계부 및 본인 회원정보만 사용

---

### ② 지출수단·소득수단 관리

- 지출수단과 소득수단은 **한 테이블**에서 **타입 구분**으로 관리한다.
- 등록
  - 공통: 수단이름, 타입(카드 또는 계좌), 사용여부(0/1, 1이면 지출·소득 등록 시 선택 가능)
  - 카드인 경우: 카드유효기간(문자) 추가 입력
- 목록 조회 (사용자별)
  - **관리 화면**: 삭제된 수단 포함
  - **지출·소득 입력**: 사용 중이고 삭제되지 않은 수단만
- 1건 상세 조회
- 1건 수정
- 삭제
  - DB에서 지우지 않고 **삭제 표시**만 함
  - 삭제 후에는 **새 지출·소득 등록 시** 그 수단을 선택할 수 없음
  - **이미 등록된 지출·소득**을 볼 때는 수단 이름이 **등록 당시 그대로** 보여야 함

---

### ③ 지출유형 관리

> 기존 '지출태그' 명칭을 **지출유형**으로 통일한다. (쇼핑, 장보기, 외식, 교육, 병원, 교통, 품위유지 등)

- 등록
  - 통계·조회 그룹 단위로 사용하는 지출 성격 분류
- 목록 조회
  - 사용자마다 기본 지출유형 데이터 제공
  - 회원가입 또는 관리자의 회원 추가 시 기본 지출유형을 DB에 자동 등록
- 1건 상세 조회
- 1건 수정
  - 활성화/비활성화
  - 유형 이름
  - 유형 아이콘 업로드
    - 저장 경로: 로컬 아이콘 저장소 / `{아이디}_{유형기본값}.{이미지확장자}`
    - 화면은 경로·파일명으로 이미지 표시
    - 기본 생성 유형의 이미지 경로는 고정
    - 사용자가 유형을 삭제해도 **이미지 파일은 삭제하지 않음**
- 1건 삭제
  - **이미 지출에 쓰인 유형은 삭제 불가** (비활성화만 가능)
  - 삭제 불가 시 사용자에게 **알림** 출력
- 지출 1건당 **지출유형 1개 필수** (다중 선택 불가)

---

### ④ 가계부 작성 (지출·소득·고정지출·엑셀)

#### 4-1. 지출·소득 작성

- 지출·소득은 사용자별로 입력
- **지출**
  - 일시불/할부 구분 (기본값: 일시불)
  - 필수 입력: 지출수단, 금액, 결제날짜, 결제장소, 결제내용, **지출유형**
  - 할부 선택 시 추가 입력: 할부 개월 수, 할부금(월 납부액), 지출 시작월(연월)
  - **할부 처리 규칙**
    - 할부 등록 시점에 해당 개월 수만큼 **지출 내역을 즉시 생성**한다. (매월 배치로 추가 생성하지 않음)
    - 목록 조회 시 **해당 월에 속하는 할부 건만** 표시
    - 수정 가능 항목: **지출 내역 단위**의 금액·결제날짜·결제장소·결제내용·지출유형 등 (일반 지출과 동일)
    - 할부 개월 수·시작월 변경은 **삭제 후 재등록**
    - **중도상환** (추가 기능): 남은 할부 지출을 삭제하여 처리
- **소득**
  - 소득수단 선택 후 소득금액·결제날짜 등 입력

#### 4-2. 지출·소득 목록 조회

- 월 단위로 사용자의 지출·소득·고정지출을 **한 목록**으로 조회
- 필터: 지출수단, 지출유형, 결제날짜 범위, 결제내용·결제장소(부분 일치 검색)
- 정렬: 결제날짜 오름/내림차순, 지출수단, 지출유형

#### 4-3. 지출·소득 수정

- 지출 1건 수정
- 소득 1건 수정

#### 4-4. 지출·소득 삭제

- 지출 1건 삭제
- 소득 1건 삭제

#### 4-5. 엑셀 일괄 등록

- `.xlsx` 형식만 지원, **최대 300행**
- 지출·소득을 구분한 정해진 양식으로 여러 건 일괄 입력
- **동일·중복 행 등록 허용** (등록 후 정리는 사용자 책임)
- 1건이라도 검증 실패 시 **전체 등록 실패**, 실패 위치·사유 응답
- 양식 다운로드 제공
  - 수단·지출유형 등 **셀에서 목록 선택** 가능
- 소득 양식 컬럼은 지출 양식과 **동일한 방식**으로 정의

#### 4-6. 고정지출

- 지출 입력과 **별도**로 관리
- **매달 지출 내역을 DB에 따로 저장하지 않음** — 목록 조회 시 해당 월에 맞게 **계산해** 지출·소득 목록과 **합쳐** 표시
- 고정지출 1건 등록
  - 고정지출 이름 (중복 허용, 변경 가능)
  - 지출수단, 금액, **결제일**(매달 몇 일), 결제내용, **지출유형**
  - **고정지출시작월** (연월, 이 달부터 목록에 표시)
  - **고정지출종료월** (연월, 이 달까지 목록에 표시)
- 고정지출 목록 조회 (페이징, 관리용)
- 고정지출 1건 상세 조회
- 고정지출 기본값 수정
  - 등록 시 값. 변경 시 **월별로 따로 수정하지 않은 달**에 적용
- 고정지출 월별 수정
  - 특정 연·월에만 금액·결제일·결제내용 등을 다르게 지정
  - 해당 월에 월별 수정이 있으면 **월별 값 우선**, 없으면 기본값
- 고정지출 1건 삭제 (붙어 있던 월별 수정 데이터도 함께 삭제)

---

### ⑤ 지출유형 목표금액 설정

- **미사용(비활성) 지출유형은 목표금액을 등록·수정할 수 없다.**
- 기본 목표금액 등록 (최소 0원, 최대 1억 원) — **사용 중인 지출유형만**
- 기본 목표금액 수정 — **사용 중인 지출유형만**
- 월별 목표금액 등록 (통계 화면, 미지정 시 기본값 사용) — **사용 중인 지출유형만**
- 월별 목표금액 수정 — **사용 중인 지출유형만**

---

### ⑥ 월별 지출·소득 통계

- 입력된 월별 지출·소득·고정지출(전개분)을 기준으로 통계 생성
- **주별 지출 합계**
  - 해당 **월 안**에서만 산출
  - 주의 시작: **월요일**
  - 해당 월 1일이 월요일이 아니면, 그 주는 **월 1일부터** 합산
- **지출유형별 지출 합계** (합계 0원인 유형은 표시하지 않음)
- **지출유형별 목표 대비 사용률**
  - 목표 100% 기준 사용 %, 초과·적정·절약 여부 표시
- **결제수단별 지출 합계** (합계 0원인 수단도 표시)
- **고정지출 vs 일반 지출 비율** 구분 통계
- **월 소득 합계** (해당 월 수입 총액)
- 통계는 **월 단위**, **저장 기능** 제공
  - 저장된 월 통계가 없으면: 현재 데이터 기준 **실시간 계산 결과** 표시
  - 저장 시: 당시 지출·소득·고정지출·목표금액을 반영해 **그 달 통계를 DB에 저장**
  - 저장된 월 통계가 있으면: **저장된 통계** 표시 (지출 수정만으로는 바뀌지 않음 — **다시 저장**해야 갱신)

## 이후 버전 (현재 범위 제외)

- OAuth·소셜 로그인 (카카오 등)
- 관리자의 타인 가계부 조회
- 할부 중도상환 전용 UI·흐름 (현재는 할부 지출 삭제로 처리 가능)
