# 1. 요구사항 정리

### ① 회원 관리 기능
- 회원가입 (id, pw, 닉네임)필수, (이메일주소, 폰번호, 소개)선택
- 로그인 로그아웃 (세션으로 로그인)ㅇ (토큰으로 로그인) 이건 나중에
- id 찾기, 비밀번호 찾기
- 회원정보 수정(pw, 닉네임, 이메일주소, 폰번호, 소개)
- 회원 목록 조회 (페이징, id 또는 닉네임으로 검색 가능해야 함)
- 회원 목록에서 등록 
- 계정에 권한 부여(일반/관리자), 기본값 관리자(1), 일반(3)
  - 관리자는 모든권한을 가진다. 일반은 회원추가를 제외한 모든 권한을 가진다.

### ② 가계부를 작성하는 기능
- 지출수단/소득수단을 등록하는 기능 (카드 또는 계좌,  지출수단이름, 카드일 경우 카드유효기간 문자로 추가입력, 사용여부 0/1로 1일경우 지출등록에서 사용가능)
- 지출수단/소득수단 목록을 조회하는 기능 (사용자별 목록)
- 지출수단/소득수단 1개의 정보를 상세조회 하는 기능
- 지출수단/소득수단 1개의 정보를 수정하는 기능
- 지출태그를 등록하는 기능(지출태그란, 쇼핑, 장보기, 외식, 교육, 병원, 교통, 품위유지 같은 지출태그를 등록해 통계로 사용하거나, 조회하기 위한 부가적인 그룹단위로 사용되기 위해 등록한다.)
- 지출태그 목록을 조회하는 기능(사용자바다 기본적인 지출태그 데이터를 제공하며, 이는 회원가입이나 회원추가시 db에 추가되어야하며 이를 통해 조회될수있어야함)
- 지출태그 1개를 수정하는 기능 (활성화 비활성화 선택 가능, 태그이름 선택가능, 태그아이콘 선택 후 업로드 가능, 태그아이콘은 로컬에 아이콘 저장소에 아이디_태그기본값.이미지확장자 로 저장하며, 프론트에서 이 경로와 이미지 이름으로 불러오게끔한다. 기본적으로 생성되는 태그는 이미지 경로가 고정이며, 사용자가 태그를 삭제하더라도 이미지는 삭제하면 안됀다.)
- 지출태그 1개를 삭제하는 기능 (지출태그 1개를 삭제한다.)
- 지출태그 1개를 조회하는 기능 (지출태그 1개를 상세조회한다.)
- 지출/소득 을 작성하는 기능.
  - 지출/소득은 사용자별로 입력하는 것이다.
  - 지출은 일시불/할부 구분해서 작성 가능해야 함.
  - 지출 입력시 지출수단을 선택해서 입력해야 함.
  - 지출 입력시 지출수단, 금액, 결제날짜, 결제장소, 결제내용, 지출태그 으로 작성한다.
  - 지출 입력시 일시불이 기본이며 할부로 선택시 몇개월 할부, 할부금 지출시작월 추가로 입력해야 한다.
  - 소득 입력시 소득수단을 선택후, 소득금액을 입력해야함.
- 지출/소득 목록을 조회하는 기능
  - 월단위 사용자의 지출/소득, 고정지출 목록을 조회한다.
  - 조회 필터 조건으로, 지출수단, 지출태그, 결제날짜 범위를 선택해서 조회가능하며, 결제내용과 결제장소를 입력해서 like검색 가능하다.
  - 결제날짜 기준 오름차순, 내림차순 정렬해서 조회 가능하다.
  - 결제수단과 지출태그 기준으로 정렬조건을 추가할 수 있다.
  - 고정지출은 어느달부터 고정지출 출력 안되도록할지 연월을 입력하는 기능이 있어, 특정월부터는 조회가 안되어야 한다
- 지출/소득 1개를 수정하는 기능(1개 에 대해 수정 가능하다.)
- 엑셀 양식을 통해서 지출/소득을 입력하는 기능
  - 여러 데이터를 한번에 일괄적으로 입력하는 기능이다.
  - 지출/소득을 구분해서 정해진 엑셀 양식대로 입력하면, 입력 가능하다.
  - 입력이 불가능한 데이터가 하나라도 존재한다면 일괄적으로 등록실패해야 하며, 어느부분에서 실패했는지 응답해야한다.
  - 엑셀에는 db값을 입력하기 불편하므로, 엑셀양식을 다운로드 하는기능을 제공해야 하며, 엑셀양식에 어떤 값을 입력해야 하는지 리스트를 제공해야 한다.(셀에서 항목마다 클릭시 콤보박스 리스트로 나와서 선택가능하도록 엑셀양식을 만들어줘야 한다.)
- 고정지출을 등록할 수 있어야 한다.
  - 고정지출은 지출 입력과는 별개로 등록한다.
  - 고정지출은 지출/소득 목록에 매월 1개씩 나와야 한다.
  - 고정지출은 결제일을 입력받아야 한다.
  - 고정지출은 지출수단, 금액, 결제날짜(결제일, 매달몇일), 결제내용, 지출태그 를 입력할수있어야한다. (지출입력처럼)
- 고정지출을 관리하기위한 고정지출 목록 조회 기능 필요하다.
  - 고정지출을 관리하기위해 사용자에게 등록된 고정지출 목록을 확인하고 선택해서 수정하기 위함이다.
  - 페이징 필요하다.
- 고정지출1개를 수정 하는 기능.
  - 고정지출은 매달마다 성격이나 금액이 조금씩 달라질 수 있으므로, 매 달마다 금액, 결제날짜, 결제내용을 수정을 통해 수정할 수 있어야 한다.
  - 기본적으로 고정지출 등록시 입력한 값이 기본적인 고정값이되며, 이 기본값도 바꿀 수 있어야 한다. (기본값 바꿀경우 다른 달에도 영향을 줄수 있다.)
  - 지출/소득 목록 조회시 지출/소득 목록조회 + 고정지출 목록 조회 를 별도로 한 뒤 합쳐서 출력하면 된다.
  - 고정지출은 어느달부터 고정지출 출력 안되도록할지 연월을 입력하는 기능을 추가해야 한다.
  - 고정지출 이름은 중복이 허용되며 변경도 가능하다.

### 지출태그의 목표금액 설정 관리
- 지출태그마다 목표금액을 입력할 수 있다. (최소0원 최대1억원)
- 지출태그마다 목표금액을 수정할 수 있다.
- 지출태그에 목표금액을 입력하는건 목표금액 기본값을 의미하며, 월마다 목표금액은 통계화면에서 월별 다르게 지정할 수 있다. (별도지정없으면 입력된 값이 목표값이 된다.)

### ⑤ 월별 지출통계
- 입력된 월별 지출 내역, 수입내역, 고정지출을 기준으로 통계를 만든다. 
  - 주별 지출합계 
  - 지출태그별 지출합계 (지출태그 합계 0원인 내역은 표시하지 않는다.)
  - 지출태그별 목표치만큼 사용한 백분율 비교 (목표치 100% 기준 합계 몇% 사용했는지, 초과했는지 적절했는지 절약했는지도 통계에 나와야 한다.)
  - 결제수단별 지출합계 (결제수단 합계 0원인 내역이 있더라도 표시한다.)
  - 지출관리 포함유무에 맞춰 해당 연-월의 분류별 지출 합계를 계산하고 저장한다.
  - 지출금액에서 고정금액 비율을 구분해서 볼 수 있도록 통계를 만들어야 한다.
- 지출통계는 기본적으로 월단위로 만들어지며, 저장기능을 제공해야 한다.
  - 저장된 월의 통계가 없으면 기본적인 통계합계를 출력한다
  - 저장기능을 통해 수정된 태그별 목표금액을 지정, 거기에 맞춰 지금기준 고정지출이나 월 지출 월 소득 기준으로 통계결과를 저장해야 한다.
  - 저장된 월의 통계자료가있으면 기본적인 자료가 아닌 저장된 지출내역을 출력해야 한다.
